# React Setup and Build

This guide covers the React frontend setup: source structure, build pipeline, linting, and testing configuration. For React architecture and component design, see [React UI Architecture](04_1_UI_react-architecture.md).

## Source Structure

- `template/js/` — React source code (entry point: `index.js`)
- `template/js/components/` — React components (`.jsx`)
- `template/js/components/__tests__/` — Jest tests for components
- `template/js/store/` — `@wordpress/data` store: actions, reducer, selectors, constants
- `template/js/store/__tests__/` — Jest tests for store actions/reducer/selectors
- `template/js/hooks/` — Reusable React hooks
- `template/js/services/` — Browser-side service modules (e.g. drag-and-drop bridge)
- `template/js/utils/` — Pure JS helpers
- `template/js/__tests__/` — Jest tests for top-level modules (e.g. `settings.js`)
- `assets/js/` — Compiled output (generated by `npm run build`, gitignored)

## Build Pipeline

**Toolchain:** `@wordpress/scripts` (webpack + Babel + ESLint + Jest).

**Commands:**
- `npm run build` — production build
- `npm start` — development build with watch mode

**Webpack mapping** (configured in `webpack.config.js`):

- `template/js/index.js` → `assets/js/foldsnap-admin.js` (sidebar bundle for the Media Library screen).
- `template/js/settings.js` → `assets/js/foldsnap-settings.js` (Settings page bundle).

## Asset Loading

`@wordpress/scripts` generates one `.asset.php` manifest per webpack entry, named after the entry. Each manifest contains:
- WordPress script dependencies (auto-detected from imports)
- A content hash for cache busting

PHP controllers enqueue the matching bundle via `wp_enqueue_script()` using the corresponding asset file.

Standalone scripts that do not depend on the React bundle (e.g. `template/js/foldsnap-dragdrop.js`) are copied — not bundled — to `assets/js/`. This is necessary because they rely on jQuery UI globals registered outside the bundle.

## Linting

**Preset:** `@wordpress/eslint-plugin/recommended` — the WordPress-curated rule set covering code quality, React best practices, accessibility, JSDoc, and Prettier formatting.

**Config file:** `.eslintrc.js`

**Commands:**
- `npm run lint` — check
- `npm run lint:fix` — auto-fix

## Testing

**Framework:** Jest with `@testing-library/react`.

**Location:** Tests live in `__tests__/` directories alongside the components they cover.

**File pattern:** `*.test.jsx` or `*.test.js`.

**Config file:** `jest.config.js`.

**Command:** `npm test`

For test writing standards (assertions, queries, user interactions), see [Test Writing Guide](99_2_APPENDIX_write_tests_guide.md).
